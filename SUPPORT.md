# Support

Support is best-effort through GitHub issues.

Use issues for:

- Bugs in analyzer behavior, reports, manifests, or the UI.
- Documentation gaps that block installation or operation.
- Focused feature requests for new recommendation rules or deployment modes.

Do not use public issues for credentials, kubeconfigs, private cluster reports, customer data, or vulnerability reports. Use [SECURITY.md](./SECURITY.md) for vulnerabilities.

Commercial support, production operations, and guaranteed response times are not provided by this repository.
